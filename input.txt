🧑‍💻 "야, 너 혹시 디스코드가 TCP만 사용하고 있는 거 아냐?"


🧑‍💻 "그렇지! TCP는 신뢰성을 보장하려고 패킷이 제대로 전달될 때까지 재전송을 반복하는데, 이 과정이 지연을 발생시켜.
 디스코드 같은 실시간 음성 통신은 약간의 데이터 손실보다 빠른 전송이 더 중요하거든!"


🧑‍💻 "UDP를 써야 해! UDP는 TCP처럼 데이터 전송을 보장하진 않지만, 지연 없이 빠르게 데이터를 보내기 때문에 실시간 
음성 채팅에 더 적합해. 그래서 디스코드는 보통 UDP를 사용해!"

동찬이는 그제야 문제의 원인을 이해했다. TCP는 신뢰성은 뛰어나지만 속도가 중요한 실시간 통신에서는 적절하지 않았다.

✅ "그럼 UDP를 활성화하면 친구들과 끊김 없이 대화할 수 있겠네!"

그렇게 동찬이는 네트워크 설정을 확인하고, 친구들과의 원활한 소통을 되찾았다. 배틀그라운드에서 팀워크가 더욱 좋아졌고, 
마침내 치킨을 먹을 수 있었다! 🍗

✅ 네트워크 장애 발생 시 체크리스트
1️⃣ 물리적 연결 확인

LAN 케이블이 정상적으로 연결되어 있는가?
스위치, 라우터, 서버가 정상적으로 작동하는가?
전원이 정상적으로 공급되고 있는가?
2️⃣ 네트워크 장비 상태 확인

스위치 및 라우터의 LED 상태를 확인했는가?
특정 장비에서만 문제가 발생하는가, 아니면 전체 네트워크에 문제가 있는가?
방화벽이나 보안 정책이 갑자기 변경되지는 않았는가?
3️⃣ IP 및 네트워크 설정 확인

PC 및 서버의 IP 주소가 정상적으로 할당되어 있는가?
DHCP 서버가 정상적으로 동작하는가?
Ping 및 Traceroute 테스트를 수행해봤는가?
4️⃣ 네트워크 트래픽 및 부하 확인

특정 장비에서 과도한 트래픽이 발생하고 있는가?
네트워크 대역폭이 가득 차 있는가?
최근에 변경된 설정이 있는가?
5️⃣ DNS 및 인터넷 연결 테스트

특정 웹사이트만 접속이 안 되는가?
내부 네트워크에서는 통신이 되지만 외부 네트워크에서만 문제가 있는가?
DNS 서버가 정상적으로 응답하고 있는가?
6️⃣ 로그 분석 및 오류 메시지 확인

라우터, 스위치, 방화벽의 로그에서 오류 메시지가 있는가?
최근 네트워크 관련 장애 이력이 있는가?
💨 “공기 순환 문제”가 네트워크 장애의 원인이 될 수 있을까?
"조민환기 좀 시켜라"의 주장이 타당하려면, 네트워크 장애가 실제로 공기 순환 문제와 관련이 있어야 한다. 
이를 위해서는 다음과 같은 추가 장비가 필요하다.

1️⃣ 서버실 및 네트워크 장비 온도 센서

네트워크 장비(스위치, 라우터, 서버 등)의 발열이 심하면 성능 저하나 다운이 발생할 수 있음.
공기 순환이 원활하지 않으면 과열로 인해 장비가 자동으로 꺼질 수도 있음.
2️⃣ 환경 모니터링 시스템

온도, 습도, 공기 흐름을 감지하는 센서를 설치하여 장비 과열 여부를 확인해야 함.
데이터센터에서는 일반적으로 CRAC(CRAC: Computer Room Air Conditioning) 시스템을 사용하여 공기 흐름을 제어함.
3️⃣ 서버 및 네트워크 장비의 온도 로그 확인

대부분의 네트워크 장비 및 서버는 온도 로그를 제공하며, 과열로 인한 장애가 발생했는지 확인할 수 있음.
예를 들어, Cisco 스위치에서는 show environment temperature 명령어를 사용하여 온도를 확인할 수 있음.

💡 무전기가 반이중(Half-Duplex) 통신을 사용하는 이유
무전기는 반이중(Half-Duplex) 통신 방식을 사용합니다. 이는 한 번에 한 사람만 말할 수 있는 방식을 의미합니다.

✅ 무전기가 반이중 통신을 사용하는 이유
1️⃣ 주파수 대역 절약

동시에 송신과 수신을 하면 두 개의 주파수(Full-Duplex)가 필요하지만,
반이중 통신은 하나의 주파수만 사용하여 주파수 대역을 절약할 수 있습니다.
2️⃣ 구조가 간단하고 저렴

Full-Duplex 통신(전화기처럼 동시 송수신)보다 회로 설계가 단순하고 비용이 저렴합니다.
3️⃣ 노이즈 및 간섭 방지

동시에 여러 사람이 말하면 혼선이 발생할 가능성이 큽니다.
한 번에 한 사람만 송신하면 명확한 의사 전달이 가능합니다.
4️⃣ 작전 수행 시 명확한 의사소통

경찰, 군대, 소방 등에서는 실수 방지를 위해 상대방이 말할 때 기다렸다가 응답하는 방식이 필요합니다.
그래서 무전기 사용자들은 "오버(Over)" 라는 말을 사용하여 송신이 끝났음을 알립니다.

📌 브라우저 렌더링 과정 요약
1⃣ HTML 파싱 → DOM 생성

HTML을 읽고 트리 구조의 DOM 생성
2⃣ CSS 파싱 → CSSOM 생성

CSS 파일을 읽고 스타일 정보가 담긴 CSSOM 생성
CSS는 렌더링 블로킹 요소 (스타일이 완성될 때까지 화면 표시 안 됨)
3⃣ Render Tree 생성

DOM + CSSOM 결합 → 화면에 표시될 요소만 포함된 렌더 트리 생성
4⃣ Layout (배치 계산)

각 요소의 크기와 위치 결정 (뷰포트 기준)
5⃣ Paint (화면 그리기)

요소를 픽셀로 변환해 색상, 그림자 등 적용
6⃣ Composition (합성 및 출력)

여러 레이어를 최적화하여 최종 화면 표시
💡 최적화 포인트
✅ 렌더링 차단 요소 최소화 → <script>에 defer 사용
✅ 리플로우 최소화 → transform 활용
✅ 페인팅 최적화 → GPU 가속 (will-change 사용)
